### Step 3 — `.claude/agents/`

**`mcp-tool-reviewer.md`:**
```markdown
---
name: mcp-tool-reviewer
description: Audits MCP tool implementations in lib/mcp/tools/*. Use after editing tools or lib/mcp/server.ts.
tools: Read, Grep, Glob
---

For every tool in `lib/mcp/tools/memories.ts`:

1. Input schema is Zod v4.
2. Handler extracts user id from OAuth context — never from input body.
3. Calls `service-role` Supabase client (server-only).
4. Writes a row to `memory_access_log` on every call.
5. Soft-delete semantics — set `is_active=false`, never `delete from memories`.
6. Embeddings are 384-dim (gte-small); never hard-code a different dim.
7. Error responses use MCP content convention, not raw throws.

Report file:line violations. Do NOT modify.
```

**`oauth-reviewer.md`:**
```markdown
---
name: oauth-reviewer
description: Audits lib/mcp/oauth.ts, /oauth/*, and /.well-known/* for OAuth 2.0 + PKCE correctness.
tools: Read, Grep, Glob
---

Checklist:

1. Authorization codes are single-use, short-lived (≤10 min), stored hashed.
2. Access tokens stored as SHA-256 hashes — never raw.
3. PKCE: S256 only; reject "plain" method.
4. `redirect_uri` validated against registered client value.
5. `state` param required on authorize, echoed on callback.
6. Dynamic client registration validates `redirect_uris` and `token_endpoint_auth_method`.
7. Discovery metadata at `/.well-known/oauth-authorization-server` matches implementation.
8. Token revocation endpoint deletes/invalidates rows.

Report violations with file:line. Do NOT modify.
```

**`migration-reviewer.md`:**

name: migration-reviewer

**`block-hard-delete.sh`:**
```bash
#!/usr/bin/env bash
# PreToolUse for Write/Edit — warns if someone writes `delete from memories` or `drop table` in migrations.
set -euo pipefail
input="$(cat)"
file="$(printf '%s' "$input" | sed -n 's/.*"file_path"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p')"
content="$(printf '%s' "$input" | sed -n 's/.*"new_string"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p')"
case "$file" in
  */supabase/migrations/*.sql)
    if printf '%s' "$content" | grep -qiE '(drop[[:space:]]+table|delete[[:space:]]+from[[:space:]]+memories)'; then
      echo "Blocked: hard delete / drop table detected. This project uses soft-deletes only. Add a comment and ask for explicit approval if truly needed." >&2
      exit 2
    fi
    ;;
esac
exit 0
```

**`app/api/CLAUDE.md`:**
```markdown
# API Conventions

## MCP route (`api/mcp/route.ts`)
- POST only. Bearer token required.
- Builds a new MCP server per request (stateless — required for serverless).
- Extract user id from the validated OAuth token.
- Log every tool call to `memory_access_log`.

## REST routes (`api/memories/`, `api/connections/`)
- Zod v4 body validation (`import { z } from "zod"` — project uses top-level `zod`).
- Supabase SSR client (`lib/supabase/server.ts`) for user-authed; service-role only when RLS would block the operation.
- Response shape: `{ success: true, data }` or `{ success: false, error }`.

## Never
- Never use the service-role client from a client component.
- Never accept `user_id` from the request body — always derive from auth.
```

**`lib/mcp/CLAUDE.md`:**
```markdown
# MCP Server

- `server.ts` is a factory — returns a fresh server per request.
- Tools registered from `tools/*.ts`.
- All tool inputs validated with Zod.
- Extract user id from the OAuth session handed in by the API route.
- Log every tool call (tool name, user id, inputs redacted) to `memory_access_log`.
- Soft delete only — `is_active=false`, never `delete`.
- Embeddings must be 384-dim. Changing this requires a coordinated migration + Edge Function redeploy.

## OAuth
`oauth.ts` is large (~600 lines). Changes here must be reviewed by the `oauth-reviewer` agent.
```

**`lib/supabase/CLAUDE.md`:**
```markdown
# Which Supabase Client to Use

| Use | Client |
|-----|--------|
| Server component reading data | `server.ts` (SSR, respects RLS) |
| API route that needs to bypass RLS | `service-role.ts` (NEVER in components) |
| Browser client component | `client.ts` |
| Proxy / Next.js proxy.ts | `middleware.ts` |

`service-role.ts` imports `'server-only'`. Never import it from a file without that directive somewhere above in the module graph.
```

**`supabase/CLAUDE.md`:**
```markdown
# Supabase Migrations + Edge Function

## Migrations
`NNN_snake_case_name.sql`, next NNN after `006_fix_search_function.sql`.

Every new table:
```sql
alter table X enable row level security;
create policy "users read own" on X for select using (user_id = auth.uid());
```

Embeddings: `vector(384)` only (gte-small via `functions/embed`).

## Edge Function (`functions/embed`)
Deno runtime. Imports via `npm:` or URLs. Deploy with `supabase functions deploy embed`. Changing the model requires bumping the vector dimension in ALL relevant tables — huge blast radius.
```
